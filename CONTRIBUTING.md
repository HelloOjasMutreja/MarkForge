# Contributing to MarkForge

Thanks for your interest in improving MarkForge.

## Development Setup

1. Install Node.js 22+ and npm.
2. Clone the repository.
3. Install dependencies:

```bash
npm install
```

4. Run desktop development mode:

```bash
npm run dev
```

## Commit Guidelines

- Use clear commit messages in imperative style.
- Keep commits focused on one logical change.
- Run lint and build checks before opening PRs.

## Pull Request Process

1. Create a feature branch from `main`.
2. Add or update tests when relevant.
3. Update documentation for behavior changes.
4. Open a PR with a clear summary and screenshots for UI updates.

## Reporting Bugs

Open a GitHub issue with:

- Environment (Windows/Linux, app version)
- Steps to reproduce
- Expected and actual behavior
- Relevant logs or screenshots
